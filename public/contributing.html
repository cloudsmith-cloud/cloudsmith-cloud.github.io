Contributing to CloudSmith

Open source from day one. Here's how to participate — now and when Phase IV opens the code repos.

Architecture & Design Phase — CloudSmith is currently in Phase I–III (architecture, research, and design). The code repos are not yet open for contributions. Phase IV is when the first code ships and we'll be ready for community contributions. Watch the GitHub org to get notified.

How we're building it

  • Architecture Decision Records (ADRs) — Every major decision has a written record explaining what was decided, why, and what alternatives were considered. No black boxes.
  • Research before decisions — We evaluate competing approaches and publish the research before committing to a direction.
  • Planning in the open — The planning repo will be publicly accessible when we enter Phase IV. ADRs, research documents, and design artifacts are all tracked on GitHub.

What you can do now

  • Watch the GitHub org — Star and watch github.com/cloudsmith-cloud to get notified when repos go public.
  • Follow the build — Architecture walkthroughs and build updates are posted at blog.cloudsmith.cloud as we progress through each phase.
  • Check the Changelog — The Changelog tracks planning milestones and architectural decisions as they're finalized.
  • Join the conversation — GitHub Discussions will open when we enter Phase IV. Early questions and ideas are always welcome.

When Phase IV opens

  • Fork → branch → PR — Standard GitHub flow. No special tooling required.
  • DCO sign-off — All commits require a Developer Certificate of Origin (DCO) sign-off. No CLA required — just a Signed-off-by: line in your commit message.
  • Module contributions — Once the Module Lifecycle Manager (MLM) spec is published, community modules can be developed independently and hosted in the cloudsmith-cloud GitHub org.
  • OEM and partner modules — Hardware vendors and integration partners can ship their own modules without forking the core platform.

License

Apache 2.0

Use it, extend it, ship it. No surprises. No proprietary lock-in. Commercial and MSP use is explicitly permitted. No contributor license agreement — just a DCO sign-off on your commits.
